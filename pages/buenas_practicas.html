Buenas practicas de accesibilidad y usabilidad

Para mejorar la experiencia de los usuarios en el sitio web de TigreVet, se han implementado heurísticas de usabilidad y accesibilidad claves. Estas estrategias buscan facilitar el acceso rápido y eficiente a la información y servicios que ofrecemos, generar una óptima experiencia de usuario y asegurar que el contenido sea accesible e inclusivo para todos los usuarios, independientemente de sus capacidades. La aplicación de estas estrategias permite a TigreVet brindar una experiencia digital completa, accesible y agradable para todos.

Usabilidad Accesibilidad

1. Visibilidad del estado del sistema

Esta heurística tiene como objetivo mantener siempre informados a los usuarios sobre qué está sucediendo en el sistema y proporcionar retroalimentación en un tiempo considerable.

  • En la barra de navegación se indicará con un color diferente el estado actual de la página que el usuario está visitando. En la sección Servicios, se destaca con color y tamaño, el servicio elegido.
  • Al programar una cita, un mensaje o indicador de "cargando" debería aparecer hasta que se confirme la reserva, y el usuario debería ver una notificación de éxito una vez finalizado el proceso.

2. Consistencia y estándares

Para que los usuarios no deban tener que adivinar el significado de los elementos del sitio, se utilizan decisiones de diseño reconocibles y consistentes en toda la interfaz.

  • Los CTAs, menús, y elementos interactivos mantienen el mismo estilo, colores y posicionamiento en todas las páginas.

3. Prevención de errores

Esta heurística implica diseñar la interfaz para minimizar los errores del usuario y, cuando ocurran, ofrecer mensajes claros y simples que indiquen cómo solucionarlos.

  • En el formulario de contacto, se validan campos para evitar errores comunes, como el ingreso incorrecto del número de teléfono, y proporciona mensajes de error específicos para ayudar al usuario a corregirlo.
  • Antes de aceptar la reserva de la cita, se mostrará un mensaje recapitulando la información y el usuario tendrá otra instancia para aceptar o corregir.

4. Minimización de la carga de memoria del usuario

El objetivo es reducir la necesidad de recordar elementos previos mostrando opciones o información relevantes al usuario en todo momento. Minimiza la carga de memoria del usuario haciendo visibles los elementos, acciones y opciones.

  • El header/menú de navegación quedará fijo en todas las páginas para que el usuario tenga al alcance las otras secciones.

5. Relación entre el sistema y el mundo real

Es lógico pensar que el diseño debe hablar el mismo idioma que los usuarios, por lo que se deben utilizar palabras y conceptos que resulten familiares al usuario, haciendo que la información tenga un orden lógico y natural.

  • En la sección de reserva de cita se incluye un calendario que sigue el diseño de los calendarios tradicionales.
  • La iconografía utilizada en el footer resuena con el mundo real, para que los usuarios reconozcan y entiendan instantáneamente.

1. Descripciones textuales alternativas

Todos los elementos visuales y multimedia deben incluir descripciones alternativas (etiquetas "alt") que permitan a los lectores de pantalla interpretarlos.

  • Todas las imágenes de mascotas, íconos de servicios, y fotos de la clínica incluyan descripciones breves y precisas para usuarios con discapacidad visual.

2. Contraste adecuado entre texto y fondo

Utilizar un contraste de color adecuado entre el texto y el fondo es esencial para que el contenido sea legible. Mejorar la legibilidad del texto asegura que el contenido sea fácil de leer para todos los usuarios, lo cual fomenta una experiencia accesible y reduce la fatiga visual, facilitando la comprensión del contenido.

  • El texto principal tiene un contraste suficiente en relación al fondo, especialmente en botones de CTA, títulos y secciones importantes.

3.Compatibilidad con tecnologías asistivas

Esta heurística se enfoca en asegurar que el sitio web sea compatible con tecnologías asistivas como lectores de pantalla. Esto implica estructurar el código y los elementos visuales de forma que estas herramientas puedan interpretar correctamente el contenido.

  • El sitio web de TigreVet cuenta con etiquetas HTML semánticas (como <header>, <nav>, <main>, <footer>) para estructurar el contenido, además de usar encabezados jerárquicos (<h1>, <h2>, etc.) en las secciones. Esto facilita que los lectores de pantalla identifiquen y naveguen por el sitio, ayudando a usuarios con discapacidad visual a moverse fácilmente entre secciones.
